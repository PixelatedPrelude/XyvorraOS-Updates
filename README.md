## ✦ So... why does this exist?

Honestly? Because I wanted to see if I could do it and I took a Linux class while working on my BA in Cloud Computing and Security. That's the whole story no a bit more.

XyvorraOS is my personal Linux distro — no team, no budget, no corporate backing. Just one person, in school, working full time in the IT industry, somehow have time to do this and a love of making things that look like they belong on a starship console. It's based on **Arch Linux** and built around a space-age aesthetic — fast, minimal, and styled like something out of a sci-fi command center.

Yes, I use AI to help debug things sometimes. But a lot of it is hand-written, tested, broken, fixed, and tested again. That's just how it goes. If something explodes, I probably know about it already. If I don't — that's what the issues tab is for.

If you're here you're either a tester, curious, or just stumbled in from the void. Either way — welcome aboard crew member.

> This repo tracks **updates and releases**. The full build source lives separately.

## ✧ Okay but what actually is it?

XyvorraOS is planned for a complete desktop experience built from scratch. Here's where things are headed and a few things we got:

| | Feature | What's the deal |
|---|---|---|
| 🎨 | **Four built-in themes** | Galactic Byzantium · Quantum Verdance · Void Pearl · Rosefire Eclipse |
| 🔐 | **Custom login screens** | Custom SDDM themes — because the first thing you see should look good |
| 🚀 | **First-boot wizard** | Walks you through themes, packages, and user setup right from the start |
| ⚙️ | **Custom installer** | Full UEFI and BIOS support — old hardware deserves love too |
| 🔄 | **Update system** | One command pulls OS updates, themes, scripts, and packages. Rollbacks included because things go wrong sometimes |
| 🖱️ | **Alt desktops** | KDE Plasma, Hyprland, Sway, and Openbox all on the list to be supported at some point |

It's a lot. It's being built one piece at a time but hey...that's fine and that makes it even more fun.

## 🔄 Updates — there will be many...oh so so many

This repo is the **official update channel** for XyvorraOS. Updates can be anything — theme tweaks, new scripts, SDDM changes, new apps, config updates, whatever needs shipping.

Once the update command is set up on your system it's just one line:

```bash
xyvorra-update
```

Something broke after an update? No panic:

```bash
xyvorra-rollback
```

All releases and changelogs live in the [Releases tab](../../releases). Go look, I try to actually write useful notes there but some of it may be jibberish

## ☄️ Crew Code — the short version

This is a chill project and the community around it should be too. There's only a handful of things that'll get you booted from the airlock:

- **No politics, religion, racism, sexisom, or bashing of other OS's** Zero. Not the place, not the time, not ever.
- **Be decent to each other.** Rude comments, personal attacks, or just being unnecessarily unpleasant — not here.
- **Keep it clean.** Bad language and that kind of vibe doesn't fit what we're building.
- **Don't take things too seriously.** Someone said something you disagree with? Cool — scroll past it. This isn't that deep no need to spiral into a blackhole over something someone online said. We're all just here because we think Linux is cool and space is cooler.

The vibe is: good people, good conversations, good project. If something bothers you, move on. If something really bothers you, flag it quietly and let me know about it. Life's too short and the galaxy's too big to get worked up over stuff on the internet.

That's it. Pretty simple crew code for a pretty simple goal — keep it fun.
